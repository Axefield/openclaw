# Analyze Argument

Identify distortions, fallacies, and weak points in arguments.

## Usage
Use this command when you need to:
- Critically analyze arguments
- Identify logical fallacies
- Find distortions or misrepresentations
- Strengthen your critical thinking
- Prepare counterarguments

## Analysis Types
- **Distortions**: Exaggeration, oversimplification, misattribution
- **Fallacies**: Strawman, ad hominem, false dichotomy, etc.
- **Weak Points**: Areas that need strengthening
- **Refutation**: Counterarguments and evidence

## Example Usage
```
Analyze this argument: "All JavaScript developers hate TypeScript because it's too complex"
Request refutation: true
Context: "Discussion about TypeScript adoption"
```

## Parameters
- **originalClaim**: The argument to analyze (required)
- **distortions**: Types of distortions to look for (optional)
- **fallacies**: Types of fallacies to identify (optional)
- **requestRefutation**: Whether to provide counterarguments (default: false)
- **context**: Additional context for analysis (optional)

## Benefits
- Improved critical thinking
- Better argument evaluation
- Identification of weak points
- Stronger counterarguments
- More objective analysis
