# Steelman Argument

Strengthen arguments by finding their most charitable, strongest version.

## Usage
Use this command when you need to:
- Understand opposing viewpoints better
- Strengthen your own arguments
- Find the best version of a claim
- Improve critical thinking
- Prepare for debates or discussions

## Steelman Process
1. Identify the strongest premises
2. Make charitable assumptions
3. Anticipate objections
4. Provide supporting evidence
5. Create the strongest possible version

## Example Usage
```
Steelman this argument: "TypeScript is unnecessary overhead for small projects"
Context: "Evaluating TypeScript adoption for a startup's MVP"
```

## Parameters
- **opponentClaim**: The argument to strengthen (required)
- **charitableAssumptions**: Assumptions that make the argument stronger (optional)
- **strongestPremises**: Key points that support the argument (optional)
- **anticipatedObjections**: Potential counterarguments (optional)
- **context**: Additional context for the argument (optional)

## Benefits
- Better understanding of opposing views
- Stronger, more robust arguments
- Improved critical thinking
- More effective communication
- Reduced confirmation bias
