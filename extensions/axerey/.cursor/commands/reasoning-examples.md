# Reasoning Examples

Practical examples demonstrating how to use the comprehensive reasoning system for real-world scenarios.

## Usage
Use this command to see practical examples of how to apply the reasoning system to common development and decision-making scenarios.

## Example Scenarios

### 1. Technology Stack Decision
```
Scenario: "Choose between React and Vue for new frontend project"

Step 1: Start reasoning session
- Task: decision-making
- Query: "Choose between React and Vue for frontend framework"
- Context: "New e-commerce platform with 5 developers"
- Memory tags: ["frontend", "react", "vue", "ecommerce"]

Step 2: Gather context
- Search memories: "react vs vue", "frontend frameworks", "team productivity"
- Get decision patterns: "framework selection outcomes"
- Retrieve context: "frontend development best practices"

Step 3: Argument analysis
- Steelman React: "Large ecosystem, extensive community, proven scalability"
- Steelman Vue: "Easier learning curve, better performance, cleaner syntax"
- Identify trade-offs: "Ecosystem vs simplicity", "Learning curve vs features"

Step 4: Decision making
- Mind balance: "Framework selection with team and project factors"
- Perspective: blend (considering both technical and business factors)
- Confidence scoring: "High confidence based on team expertise and project requirements"

Step 5: Learning integration
- Store decision: "Chose React for ecosystem and team expertise"
- Label outcome: "Success - team productivity improved"
- Extract rules: "Choose framework based on team expertise and ecosystem needs"
```

## Reasoning Patterns

### Technical Decision Pattern
```
1. Define technical requirements and constraints
2. Search for similar past decisions and outcomes
3. Analyze technical trade-offs objectively
4. Apply mind-balance for probabilistic evaluation
5. Store decision rationale and track outcomes
```

### Strategic Analysis Pattern
```
1. Gather comprehensive context from memories
2. Apply multi-perspective analysis (technical, business, risk)
3. Use steelman to strengthen different strategic options
4. Synthesize insights into coherent strategy
5. Extract strategic rules for future planning
```

### Problem Solving Pattern
```
1. Search memories for similar problems and solutions
2. Apply systematic debugging approach
3. Evaluate solutions with risk and impact analysis
4. Implement solution with outcome tracking
5. Extract problem-solving rules for future use
```

## Best Practices for Reasoning

### Context Integration
- **Always search memories first** before external research
- **Use context broker** for task-specific information
- **Consider decision patterns** from past outcomes
- **Apply learned rules** and procedures
- **Store new insights** for future reasoning

### Analysis Quality
- **Use appropriate reasoning tools** for the task type
- **Apply multiple perspectives** systematically
- **Consider uncertainty and risk** explicitly
- **Generate honest confidence scores**
- **Document reasoning process** for learning

### Learning & Adaptation
- **Track reasoning outcomes** systematically
- **Extract rules** from successful reasoning
- **Learn from failures** and suboptimal choices
- **Adapt techniques** to different contexts
- **Build reasoning expertise** over time

## Common Reasoning Mistakes to Avoid

### Insufficient Context
- Not searching memories before reasoning
- Ignoring past similar decisions and outcomes
- Failing to consider relevant constraints and factors

### Biased Analysis
- Not steelmanning opposing viewpoints
- Confirmation bias in evidence selection
- Overconfidence in reasoning quality

### Poor Learning Integration
- Not storing reasoning process and outcomes
- Failing to extract rules from successful patterns
- Not tracking outcomes for continuous improvement

### Inadequate Documentation
- Not documenting reasoning rationale
- Failing to store decision context
- Missing confidence scoring and uncertainty handling
