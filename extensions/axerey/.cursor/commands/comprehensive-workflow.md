# Comprehensive Workflow

Complete workflow combining multiple Vagogon tools for complex tasks and projects.

## Usage
Use this command for comprehensive project work that requires:
- Multi-step problem solving
- Knowledge accumulation and organization
- Decision making with reasoning
- Learning and adaptation
- Long-term project management

## Workflow Phases

### 1. Project Initiation
```
1. Start session with clear goals
2. Set up persona for project type
3. Gather initial context and requirements
4. Record initial plan with source: 'plan'
5. Store project parameters and constraints
```

### 2. Research & Learning
```
1. Search for relevant existing knowledge
2. Record research findings with source: 'execution'
3. Store new learnings and insights
4. Consolidate related information into beliefs
5. Extract patterns and rules
```

### 3. Planning & Decision Making
```
1. Use mind-balance for major decisions
2. Steelman different approaches
3. Get context for decision support
4. Record decision reasoning with reasoning tags
5. Store decision rationale and outcomes
```

### 4. Execution & Monitoring
```
1. Get task-specific context from context broker
2. Execute with memory guidance
3. Record execution steps with source: 'execution'
4. Label outcomes and results
5. Grade context helpfulness
```

### 5. Reflection & Learning
```
1. Reflect on lessons learned
2. Record consolidated beliefs (belief: true)
3. Extract procedural rules (type: 'procedural')
4. Mark important lessons (pinned: true)
5. Consolidate project memories
6. Create daily summaries
```

## Example Workflows

### New Technology Implementation
```
1. Start session: "Implement TypeScript in existing JavaScript project"
2. Switch to technical persona
3. Record initial plan: "Gradual TypeScript migration with strict mode"
   - Source: 'plan'
   - Tags: ["typescript", "migration", "planning"]
4. Search memories: "TypeScript migration", "JavaScript conversion"
5. Record research findings: "TypeScript migration best practices"
   - Source: 'execution'
   - Tags: ["typescript", "research", "migration"]
6. Mind balance: "TypeScript vs gradual migration approach"
7. Record decision reasoning: "Chose gradual migration for team adoption"
   - Tags: ["typescript", "reasoning", "decision"]
8. Store decision: "Use gradual migration with strict mode"
9. Execute implementation with context guidance
10. Record execution steps: "Successfully implemented TypeScript strict mode"
    - Source: 'execution'
    - Outcome: success
    - Score: 0.85
11. Label outcomes: success/failure with metrics
12. Record belief: "TypeScript strict mode prevents runtime errors"
    - Belief: true
    - Type: semantic
13. Extract rules: "TypeScript migration best practices"
    - Type: 'procedural'
14. Mark important: "TypeScript configuration is critical for code quality"
    - Pinned: true
15. End session with summary
```

## Workflow Benefits
- **Systematic Approach**: Structured process for complex tasks
- **Knowledge Accumulation**: Builds expertise over time
- **Learning Integration**: Learns from each execution
- **Context Awareness**: Uses relevant memories for guidance
- **Adaptive Improvement**: Gets better with each workflow execution

## Best Practices
1. **Always start with sessions** for focused work
2. **Use appropriate personas** for different task types
3. **Search before starting** to avoid reinventing solutions
4. **Store decisions and learnings** throughout the process
5. **Grade contexts and label outcomes** for learning
6. **Reflect and consolidate** at the end of each phase
7. **Extract rules** from successful patterns
8. **Use reasoning tools** for complex decisions
