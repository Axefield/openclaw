# Troubleshooting Guide

Systematic problem-solving using memory context and reasoning tools.

## Usage
Use this command when you encounter:
- Bugs or errors in code
- Performance issues
- Integration problems
- Configuration issues
- Unexpected behavior

## Troubleshooting Process
1. **Problem Analysis**: Understand the issue clearly
2. **Record Troubleshooting Plan**: Document debugging approach and steps
3. **Context Retrieval**: Find similar problems and solutions
4. **Pattern Matching**: Identify common problem patterns
5. **Solution Evaluation**: Use reasoning tools to evaluate options
6. **Record Solution Process**: Document debugging steps and findings
7. **Record Solution Outcome**: Store solution with effectiveness metrics
8. **Extract Troubleshooting Rules**: Create procedural knowledge from successful solutions
9. **Documentation**: Store the solution for future reference

## Example Usage
```
Troubleshooting:
Problem: "TypeScript compilation errors in production build"
Context: "Next.js project with strict TypeScript config"
Error: "Type 'any' is not assignable to type 'string'"

1. Record troubleshooting plan: "Debug TypeScript compilation errors in production"
   - Source: 'plan'
   - Tags: ["troubleshooting", "typescript", "compilation", "planning"]

2. Get context from context broker for troubleshooting

3. Execute systematic debugging approach

4. Record solution process: "Identified missing type annotations in API response"
   - Source: 'execution'
   - Tags: ["troubleshooting", "typescript", "solution"]
   - Outcome: success

5. Record solution outcome: "Fixed TypeScript errors by adding proper type annotations"
   - Source: 'execution'
   - Outcome: success
   - Score: 0.9
   - Efficiency: 0.85

6. Record belief: "TypeScript strict mode catches type mismatches that cause runtime errors"
   - Belief: true
   - Type: semantic

7. Extract rule: "Always add proper type annotations for API responses in TypeScript"
   - Type: 'procedural'

8. Mark important: "TypeScript compilation errors must be fixed before deployment"
   - Pinned: true
```

## Troubleshooting Steps
1. **Reproduce**: Confirm the problem consistently
2. **Isolate**: Narrow down the scope
3. **Research**: Check memory for similar issues
4. **Hypothesize**: Generate potential solutions
5. **Test**: Try solutions systematically
6. **Document**: Record the solution
