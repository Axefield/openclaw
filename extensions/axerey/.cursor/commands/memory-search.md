# Search Memories

Find relevant memories using semantic search or specific queries.

## Usage
Use this command when you need to:
- Find previous solutions or decisions
- Look up project context
- Retrieve coding patterns
- Access meeting notes or discussions
- Find related information across sessions

## Search Types
- **Semantic Search**: Find conceptually similar memories
- **Query Search**: Find memories matching specific terms
- **Tag Filtering**: Search within specific categories
- **Recent Memories**: Get latest memories

## Example Usage
```
Search for memories about: "TypeScript configuration"
Tags: typescript, config
Limit: 5
```

## Parameters
- **query**: Search terms or semantic description (required for semantic search)
- **tags**: Filter by specific tags (optional)
- **limit**: Number of results (1-50, default: 8)
- **sessionId**: Search within specific session (optional)

## Tips
- Use descriptive queries for better semantic matching
- Combine with tags for more precise results
- Use recent memories for current context
- Search before starting new tasks to avoid duplication
