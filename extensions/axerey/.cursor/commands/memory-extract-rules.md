# Extract Rules

Pattern mine execution records to generate IF/THEN procedural rules from successful outcomes.

## Usage
Use this command when you need to:
- Discover successful patterns from past executions
- Generate procedural rules automatically
- Learn from successful outcomes
- Create decision-making guidelines
- Build knowledge base of best practices

## Rule Extraction Process
1. **Pattern Analysis**: Analyze execution records for common patterns
2. **Outcome Correlation**: Correlate patterns with successful outcomes
3. **Rule Generation**: Create IF/THEN rules with confidence scores
4. **Support Filtering**: Filter rules by minimum occurrence threshold
5. **Validation**: Validate rules against win-rate analysis

## Example Usage
```
Extract rules:
Tag: "typescript-development"
Min support: 3 occurrences
Focus: "successful patterns"
```

## Rule Types
- **IF/THEN Rules**: Conditional procedural knowledge
- **Pattern Rules**: Recurring successful patterns
- **Decision Rules**: Decision-making guidelines
- **Process Rules**: Workflow optimization rules

## Parameters
- **tag**: Specific tag to analyze (optional)
- **minSupport**: Minimum occurrences to consider (default: 3)

## Benefits
- **Automatic Learning**: Discovers patterns without manual analysis
- **Procedural Knowledge**: Creates actionable rules from experience
- **Success Optimization**: Focuses on patterns that lead to wins
- **Knowledge Accumulation**: Builds institutional knowledge over time
- **Decision Support**: Provides rule-based decision guidance

## Generated Rule Format
```
IF [condition] THEN [action] (confidence: 0.85, support: 5)
```

## Best Practices
- Use specific tags for focused rule extraction
- Set appropriate minimum support thresholds
- Regularly extract rules from recent executions
- Review and validate generated rules
- Use rules to guide future decisions
