# Reflect and Learn

Write distilled lessons and update procedural rules based on experience.

## Usage
Use this command when you want to:
- Capture key learnings from experiences
- Update procedural knowledge
- Create actionable rules
- Document best practices
- Improve future performance

## Reflection Process
1. Analyze recent experiences and outcomes
2. Identify key patterns and lessons
3. Distill actionable insights
4. Update procedural rules
5. Create guidelines for future use

## Example Usage
```
Reflect on:
Topic: "TypeScript project setup"
Outcome: "Successfully implemented strict typing with no build errors"
Notes: "Using strongly mapped interfaces instead of multiple specific methods improved maintainability"
```

## Parameters
- **topic**: Subject of reflection (required)
- **outcome**: What was achieved or learned (required)
- **notes**: Additional insights or details (optional)

## Benefits
- Learning from experience
- Procedural rule updates
- Best practice documentation
- Improved future performance
- Knowledge distillation
