# Pin Set Management

Create curated memory sets for quick access and organized knowledge management.

## Usage
Use this command when you need to:
- Organize related memories into sets
- Create quick access collections
- Group memories by topic or project
- Build reference collections
- Organize knowledge systematically

## Pin Set Operations
- **Create Set**: Group memories into a named collection
- **Access Set**: Quickly retrieve all memories in a set
- **Manage Sets**: Add/remove memories from sets
- **Organize Knowledge**: Structure information hierarchically

## Example Usage
```
Create pin set:
Name: "TypeScript Best Practices"
Memory IDs: ["mem_123", "mem_456", "mem_789"]

Access pin set: "TypeScript Best Practices"
```

## Set Types
- **Topic Sets**: Memories grouped by subject matter
- **Project Sets**: Memories related to specific projects
- **Process Sets**: Memories about workflows and procedures
- **Reference Sets**: Quick reference collections
- **Learning Sets**: Educational content collections

## Parameters
- **name**: Name for the pin set (required)
- **ids**: Array of memory IDs to include (required)

## Benefits
- **Quick Access**: Instant access to related memories
- **Organization**: Structured knowledge management
- **Context Preservation**: Maintains relationships between memories
- **Efficiency**: Reduces search time for related information
- **Knowledge Structuring**: Creates hierarchical knowledge organization

## Best Practices
- Use descriptive set names
- Group logically related memories
- Keep sets focused and manageable
- Regular review and maintenance
- Use consistent naming conventions
