# Decision Analysis

Comprehensive decision-making system that integrates memory, reasoning, and outcome analysis for optimal choices.

## Usage
Use this command when you need to:
- Make complex decisions with multiple factors
- Analyze decision options systematically
- Consider past outcomes and patterns
- Apply probabilistic reasoning to choices
- Build decision-making expertise over time

## Decision Analysis Process

### 1. Decision Context Setup
```
1. Start decision session with clear objectives
2. Set up appropriate analytical persona
3. Use context broker to get decision context
4. Define decision criteria and constraints
5. Establish success metrics and evaluation criteria
```

### 2. Option Generation & Analysis
```
1. Generate comprehensive list of options
2. Use memory search for similar past decisions
3. Use decision patterns to analyze outcomes
4. Use steelman to strengthen each option objectively
5. Identify pros, cons, and trade-offs
```

### 3. Probabilistic Evaluation
```
1. Use mind-balance for probabilistic analysis
2. Apply angel/demon perspectives for balance
3. Consider risk factors and uncertainty
4. Evaluate confidence levels for each option
5. Generate probability-weighted recommendations
```

### 4. Decision Implementation & Learning
```
1. Make decision with confidence scoring
2. Store decision rationale and process
3. Track implementation and outcomes
4. Label results for learning system
5. Extract decision rules for future use
```

## Decision Analysis Parameters

### Core Parameters
- **decision_type**: technical, strategic, architectural, process, business
- **decision_question**: The specific decision to be made
- **context**: Background and constraints for decision
- **criteria**: Evaluation criteria and success metrics
- **memory_tags**: Tags for relevant memory search

### Analysis Parameters
- **analysis_depth**: surface, moderate, comprehensive, exhaustive
- **perspective_mode**: angel, demon, blend, probabilistic
- **confidence_threshold**: Minimum confidence for recommendations
- **memory_limit**: Number of memories to consider (1-20)
- **learning_mode**: Store decision process for future learning

### Advanced Parameters
- **risk_tolerance**: Risk appetite for decision (0-1)
- **time_horizon**: Decision impact timeframe
- **stakeholder_impact**: Who is affected by decision
- **reversibility**: How easily decision can be changed
- **success_metrics**: How to measure decision success

## Decision Quality Framework

### Information Quality
- **Completeness**: All relevant factors considered
- **Accuracy**: Information is current and reliable
- **Relevance**: Information directly relates to decision
- **Bias Detection**: Identified and addressed biases
- **Uncertainty Handling**: Proper treatment of unknowns

### Analysis Quality
- **Systematic Approach**: Structured analysis process
- **Multiple Perspectives**: Considered different viewpoints
- **Trade-off Analysis**: Clear understanding of trade-offs
- **Risk Assessment**: Proper evaluation of risks
- **Confidence Scoring**: Honest assessment of certainty

### Decision Quality
- **Alignment**: Decision aligns with objectives
- **Feasibility**: Decision is implementable
- **Timeliness**: Decision timing is appropriate
- **Reversibility**: Consideration of decision reversibility
- **Success Metrics**: Clear measures of success
