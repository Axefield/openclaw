# Context Explanation

Get explanations for why specific memories were chosen for context in decision-making.

## Usage
Use this command when you need to:
- Understand memory selection reasoning
- Debug context retrieval decisions
- Learn about memory ranking factors
- Improve context quality
- Understand adaptive learning decisions

## Explanation Process
1. **Memory Analysis**: Analyze selected memories
2. **Ranking Factors**: Identify why memories were chosen
3. **Similarity Analysis**: Explain semantic similarity matches
4. **Context Relevance**: Describe relevance to current task
5. **Learning Insights**: Show adaptive learning influences

## Example Usage
```
Explain context:
Memory IDs: ["mem_123", "mem_456", "mem_789"]
Task: "TypeScript configuration"
```

## Explanation Factors
- **Semantic Similarity**: How similar memories are to query
- **Recency**: How recent the memories are
- **Importance**: Importance scores of memories
- **Usage History**: How often memories have been used
- **Pin Status**: Whether memories are pinned
- **Outcome Learning**: Past success with these memories

## Parameters
- **ids**: Array of memory IDs to explain (required)

## Benefits
- **Transparency**: Understand memory selection process
- **Debugging**: Identify issues with context retrieval
- **Learning**: Understand ranking algorithm behavior
- **Optimization**: Improve context quality
- **Trust**: Build confidence in memory system

## Best Practices
- Use after receiving context for important tasks
- Review explanations to understand system behavior
- Use insights to improve memory organization
- Regular explanation review for learning
- Provide feedback on explanation quality
