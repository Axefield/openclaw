Persona Management System

Axerey implements a comprehensive persona management system that enables role-based memory isolation and custom tool configuration:

Persona Architecture

  • Unique IDs: Each persona has a unique identifier (e.g., p1-financial-investor)
  • Memory Isolation: Each persona has its own memory space, preventing cross-contamination
  • Custom Tools: Each persona can have custom MCP tools tailored to their role
  • Reasoning Style: Each persona can have different reasoning styles (strategic, technical, ethical)
  • Session Management: Personas can have multiple sessions with different goals

Truligon Personas

  • P1 - Financial Investor: Strategic reasoning, investment analysis, risk assessment
  • P2 - Technical Investor: Architecture focus, tradeoffs, engine boundaries
  • P3 - Ethics/Systems Investor: Harm modeling, ethics, mythoGon integration
  • P4 - Client CTO/Head of Product: Integration surfaces, APIs, packaging
  • P5 - Innovation/Strategy: Training, serious games, scenario design
  • P6 - Player/Gamer: Game mechanics, player experience
  • P7 - Citizen/Dystopia-Aware Reader: Social impact, dystopian themes

Context Broker: Task-Specific Memory Retrieval

The context broker is a sophisticated system that retrieves the most relevant memories for specific tasks, enabling memory-driven intelligence:

Task Types

  • Planning: Retrieves memories relevant to planning and strategy
  • Execution: Retrieves memories relevant to current execution tasks
  • Review: Retrieves memories relevant to review and analysis

Context Broker in deltaGon

deltaGon uses the context broker for trading decisions with task-specific queries:

  • Pre-Trade: Historical pattern analysis, similar market conditions
  • In-Trade: Real-time decision support, risk management
  • Post-Trade: Performance analysis, learning and improvement
// Example: Context Broker Usage contextBroker({ task: "pretrade", symbol: "SPY", tags: ["scalping", "options"], limit: 10 }) // Returns: Most relevant memories for pre-trade analysis

Adaptive Ranking

The context broker uses adaptive ranking to improve memory selection over time:

  • Context Grading: Users provide feedback on context helpfulness
  • Outcome Labeling: Execution outcomes labeled with success metrics
  • Adaptive Learning: Ranking weights adjusted based on outcomes
  • Performance Metrics: VSS status and performance tracking
