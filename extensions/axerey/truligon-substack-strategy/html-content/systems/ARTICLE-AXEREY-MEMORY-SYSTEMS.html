Memory Types: Episodic, Semantic, Procedural

Axerey implements three distinct memory types, each serving different purposes in AI applications:

Episodic Memory

Purpose: Stores specific events and experiences with temporal context. Use Cases: Trade history (deltaGon), user interactions, session data, specific occurrences.

  • Structure: Event-based with timestamps, context, outcomes
  • Example: "11/20/2025: SPY options trade, +52% in <1 hour, ascending triangle pattern"
  • Investment Value: Creates switching costs (trading history locked in Axerey)

Semantic Memory

Purpose: Stores facts, concepts, and knowledge without temporal context. Use Cases: PineScript knowledge (car02/PAWS), domain knowledge, conceptual understanding.

  • Structure: Fact-based with relationships, concepts, knowledge
  • Example: "PineScript syntax patterns, trading strategies, technical indicators"
  • Investment Value: Knowledge base creates lock-in, network effects improve with usage

Procedural Memory

Purpose: Stores rules, procedures, and how-to knowledge. Use Cases: Trading rules, decision-making procedures, automated workflows.

  • Structure: Rule-based with conditions, actions, outcomes
  • Example: "If ascending triangle pattern detected, wait for 3-4 tests before breakout"
  • Investment Value: Automated learning creates defensible moats

MCP Tools: 32+ Tools via Model Context Protocol

Axerey provides 32+ MCP tools for AI agent integration, enabling memory-driven intelligence across applications:

Memory Operations

  • memory-store: Store new memories with text, tags, importance, expiration
  • memory-search: Semantic search through stored memories
  • memory-recall: Retrieve memories by query or get recent ones
  • memory-update: Modify existing memory text
  • memory-pin: Pin/unpin memories for quick access
  • memory-forget: Delete a memory by ID

Advanced Memory Management

  • memory-consolidate: Cluster memories and produce semantic beliefs
  • memory-extract-rules: Pattern mine executions vs outcomes to emit procedural rules
  • memory-reflect: Write a distilled lesson and update procedural rules
  • memory-decay: Maintain memory scores and delete/archive stale items
  • memory-pin-set: Create curated memory sets for quick access

Context & Learning

  • context-broker: Returns exact set of memories for specific tasks
  • context-grading: Provide feedback on context helpfulness for adaptive ranking
  • outcome-labeling: Label execution outcomes with success metrics
  • context-explanation: Explain why specific memories were chosen
  • adaptive-learning: Adaptive learning system improvements
